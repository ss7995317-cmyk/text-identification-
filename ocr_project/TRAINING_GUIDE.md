# 🏋️ YOLO Training Guide — Custom Text Detection
## How to Train YOLOv11n on New Text / Classes

---

## 📋 Overview

This project uses **YOLOv11n** for text region detection and **EasyOCR/Tesseract** for reading the text.  
Training YOLO teaches it *where* the text is on the image (bounding boxes).  
EasyOCR/Tesseract then *reads* those regions.

---

## 🗂️ Step 1 — Collect Images

Put your images inside the project folder:

```
training/
  images/
    train/   ← 80% of your images go here (e.g. 800 images)
    val/     ← 20% of your images go here  (e.g. 200 images)
  labels/
    train/   ← YOLO label .txt files (auto-generated by labeling tool)
    val/     ← YOLO label .txt files
```

**Tips for good training data:**
- Minimum: **100 images** per class (more = better; 500+ recommended)
- Include variety: different fonts, sizes, lighting, angles, backgrounds
- For printed documents: scan at 150–300 DPI
- Include both easy and hard examples

---

## 🏷️ Step 2 — Label Your Images

You need to draw bounding boxes around text regions.  
Use one of these free tools:

### Option A — LabelImg (Desktop, Recommended)
```bash
pip install labelImg
labelImg
```
1. Open LabelImg → File → Open Dir → select `training/images/train/`
2. Set Save Dir → `training/labels/train/`
3. Set format to **YOLO** (top-left dropdown)
4. Draw boxes around each text region → press `W` to draw
5. Label each box with your class name (e.g. `text`)
6. Save with `Ctrl+S`

### Option B — Roboflow (Web, Easiest)
1. Go to https://roboflow.com → Create project → Object Detection
2. Upload your images
3. Use the web UI to draw boxes
4. Export as **YOLO v8 format**
5. Download and place in `training/` folder

### Option C — CVAT (Web, Advanced Teams)
1. Go to https://cvat.ai
2. Create task, upload images
3. Annotate, export as YOLO format

---

## 📝 Label File Format (YOLO)

Each image needs a `.txt` file with the same name in `training/labels/train/`:

```
# format: class_id center_x center_y width height  (all normalized 0–1)
0 0.512 0.342 0.450 0.085
0 0.512 0.500 0.600 0.090
```

- `0` = class ID (0 for first class, 1 for second, etc.)
- All values are **normalized** (divided by image width/height)

---

## ⚙️ Step 3 — Configure Classes

Edit `training/dataset.yaml`:

```yaml
# Single class (detect all text as one category)
nc: 1
names: ['text']

# Multiple classes (distinguish text types)
nc: 3
names: ['printed_text', 'handwritten_text', 'table_header']
```

Also update `train_yolo.py` → `TRAINING_CONFIG` if needed.

---

## 🚀 Step 4 — Run Training

```bash
# Basic training (default: 100 epochs, single 'text' class)
python train_yolo.py

# Custom classes
python train_yolo.py --classes printed_text handwritten_text

# More epochs for better accuracy
python train_yolo.py --epochs 200

# Use GPU (much faster — NVIDIA only)
python train_yolo.py --device 0

# Smaller batch if GPU memory error
python train_yolo.py --batch 8 --device 0
```

Training progress will be printed to console.  
Best model is automatically saved to `models/yolo11n_text.pt` when done.

---

## 📊 Step 5 — Monitor Training

During training, check `runs/detect/text_detector/`:

```
runs/detect/text_detector/
  weights/
    best.pt       ← USE THIS (best validation accuracy)
    last.pt       ← last epoch checkpoint
  results.csv     ← loss & mAP per epoch
  results.png     ← training curves plot
  confusion_matrix.png
  val_batch0_pred.jpg  ← visual predictions on validation set
```

**Key metrics to watch:**
| Metric | Good Value |
|--------|-----------|
| mAP50 | > 0.85 |
| Precision | > 0.85 |
| Recall | > 0.80 |
| Box Loss | Decreasing |

---

## 🔄 Step 6 — Use Your Trained Model

After training completes, the script auto-copies the model:

```bash
# Model is saved automatically to:
models/yolo11n_text.pt

# Restart the Flask app to use new model:
python app.py
```

To manually copy:
```bash
cp runs/detect/text_detector/weights/best.pt models/yolo11n_text.pt
```

---

## 🔁 Step 7 — Improve Accuracy (Iterative Training)

Training is iterative. To improve:

1. **Add more images** — especially images where the model fails
2. **Fix bad labels** — review `runs/.../val_batch*_pred.jpg`
3. **Increase epochs** — try 200–300 for better convergence
4. **Data augmentation** — already enabled in `TRAINING_CONFIG`
5. **Fine-tune hyperparameters** in `TRAINING_CONFIG`

```python
# In train_yolo.py, TRAINING_CONFIG — key settings to tune:
'epochs': 200,      # More epochs = better (up to diminishing returns)
'batch': 32,        # Larger batch = more stable (if GPU allows)
'lr0': 0.005,       # Lower LR for fine-tuning a pretrained model
'imgsz': 1280,      # Larger image size = better for small text
```

## ❓ Troubleshooting

| Problem | Solution |
|---------|----------|
| `No module named 'ultralytics'` | `pip install ultralytics` |
| `No module named 'easyocr'` | `pip install easyocr` |
| CUDA out of memory | Reduce `--batch 8` or use `--device cpu` |
| mAP not improving | Add more diverse images, fix labels |
| Text not detected | Lower confidence threshold in `ocr_engine.py` → `if conf > 0.1` |
| Tesseract not found | Install Tesseract binary: https://github.com/tesseract-ocr/tesseract |
